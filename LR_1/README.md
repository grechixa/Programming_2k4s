# README — Итераторы чисел Фибоначчи в Python

## Описание

Реализованы два способа создания итератора последовательности Фибоначчи:

1. Упрощённый — через метод `__getitem__`
2. Классический — через методы `__iter__` и `__next__`

Оба варианта позволяют использовать объект в цикле `for`.

---

## Теория

### Как Python перебирает объекты

Python умеет итерироваться по объектам двумя способами.

#### Способ 1 — итераторный протокол

Объект считается итератором, если реализует:

* `__iter__()` — возвращает объект-итератор
* `__next__()` — возвращает следующий элемент

Когда элементы заканчиваются, метод `__next__()` обязан вызвать:

```
StopIteration
```

---

#### Способ 2 — доступ по индексам

Если метода `__iter__` нет, Python пытается перебирать объект так:

```
obj[0]
obj[1]
obj[2]
...
```

Перебор продолжается до тех пор, пока не возникнет ошибка:

```
IndexError
```

---

## Реализация №1 — через `__getitem__`

```python
class FibonacciSimple:
    def __init__(self, n):
        self.n = n

    def __getitem__(self, index):
        if index >= self.n:
            raise IndexError

        a, b = 0, 1
        for _ in range(index):
            a, b = b, a + b
        return a
```

### Принцип работы

* Python сам увеличивает индекс
* каждый раз вызывает `__getitem__`
* перебор прекращается при `IndexError`

Особенность: состояние не хранится — каждый элемент вычисляется заново.

---

## Реализация №2 — классический итератор

```python
class FibonacciIter:
    def __init__(self, n):
        self.n = n

    def __iter__(self):
        self.a = 0
        self.b = 1
        self.i = 0
        return self

    def __next__(self):
        if self.i >= self.n:
            raise StopIteration

        value = self.a
        self.a, self.b = self.b, self.a + self.b
        self.i += 1
        return value
```

### Принцип работы

* `__iter__` запускает итерацию и подготавливает начальные значения
* `__next__` возвращает следующий элемент
* при достижении лимита вызывается `StopIteration`

---

## Разница подходов

| Способ                  | Есть состояние | Производительность | Кто управляет |
| ----------------------- | -------------- | ------------------ | ------------- |
| `__getitem__`           | нет            | ниже               | Python        |
| `__iter__` + `__next__` | да             | выше               | программист   |

---

## Частые ошибки

Неправильно:

* возвращать `None` вместо `StopIteration`
* забывать остановку цикла
* хранить состояние итерации в `__init__`

Правильно:

* состояние создаётся в `__iter__`
* завершение через исключения

---

## Мини-шпаргалка

* `__iter__` — запуск перебора
* `__next__` — следующий элемент
* `StopIteration` — конец итерации
* `IndexError` — конец последовательности

---

## Пример использования

```python
for x in FibonacciIter(7):
    print(x)
```

Вывод:

```
0 1 1 2 3 5 8
```

---

## Итог

* `__getitem__` — простой способ реализовать перебор
* `__iter__` + `__next__` — полноценный итератор
* генераторы (`yield`) — сокращённая форма записи итераторов
